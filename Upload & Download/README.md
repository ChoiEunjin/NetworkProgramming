< 파일 전송 서버와 클라이언트 >

1. 클라이언트는 홈 디렉토리, 서버는 홈의 ‘server’ 폴더에 file_server.c라는 이름으로 저장되어 있다. 이는 클라이언트와 서버의 실행 디렉토리를 다르게 하기 위함이다.

2. 클라이언트 실행 시 IP 주소와 포트 번호와 함께 ‘put’이라는 단어를 입력하면 서버에 전송하고 싶은 파일명을 입력하도록 한다. 파일명 입력시 서버가 실행되는 디렉토리에 입력한 파일이 업로드된다.

3. 클라이언트 실행 시 IP 주소와 포트 번호와 함께 ‘get’이라는 단어를 입력하면 현재 서버 디렉토리의 파일을 출력해주며 서버로부터 받고 싶은 파일명을 입력하도록 한다. 파일명을 입력한 후에는 파일을 저장하고 싶은 디렉토리를 입력할 수 있으며 입력한 디렉토리에 파일이 다운로드된다.

4. 바이너리 파일은 파일명에 “binary”라는 단어를 명시하여 별도의 처리 과정을 사용한다.
